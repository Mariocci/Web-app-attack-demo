SQL Injection — tautology demo

If the checkbox Turn vulnerability on is checked, the backend will be queried using the vulnerable route (string concatenation). Enter a username and password and press Submit. When the vulnerability is enabled, a payload such as: 

' OR 1=1 --

in the password field will return all users from the database.

SQL Injection Examples

  • Tautology: ' OR 1=1 --
  • Illegal/logically incorrect query: ' AND 1=2; --
  • Blind SQL injection: ' OR username='alice' --
  • Union query: ' UNION SELECT 1, 'hacker', 'hacker@example.com', 'secret' --
  • Stacked queries: '; DROP TABLE users; --
